Semantic Elements

What are Semantic Elements?

Semantic Elements are used because they have a define the context, of what is contained between their opening and closing tags. Non-Semantic Elements include <div> and <span> elements. These elements do not give any hint as to what will be contained between their opening and closing tags. They simply change the way in which the content is presented.

Benefits of using Semantic Elements:

  • Better Search Engine Optimization
  • Easy to understand for other developers
  • Easier to read for screenreaders, which enabled the visually impaired to have access to your content

Attributes

Some elements allow for what is called an attribute. An attribute provides additional information about an element. For example if you are considering placing an image on your page, you will have to use the <img> tag. Its a self-closing tag, but you will need to include the src= attribute in the opening tag. This enables you to include the address of the image you are trying to add to your page. All attributes are placed in the opening tag of an element. An attribute keyword is usually followed by an equals = sign, which then lists any values the attribute requires, like a name or sizing info. Other attributes include id and class, which are CSS selectors. There are also height and width attributes that allow you to resize your image. attributes.
